# 피보나치 알고리즘 성능 프로파일러

|입력값 f(n)|시간|
|------|---|
|10|2.932초|
|20|3.592초|
|30|3.11초|
|40|15.989초|

## 알고리즘 로직

$f(n) = f(n-1) + f(n-2)$ (단, $f(0)=0, f(1)=1$)

## 시간복잡도

* **Big-O:** $O(2^n)$ (정확히는 황금비와 연관된 $O(1.618^n)$)
* **분석 이유:**
    * 함수가 한 번 호출될 때마다 다시 두 번의 호출이 발생하는 **이진 트리** 구조를 가집니다.
    * 트리의 높이가 $n$인 경우, 전체 노드의 수는 대략 $2^n$에 비례합니다.
    * **중복 계산:** $f(n-2)$ 등의 하위 연산이 반복적으로 수행되어 $n$이 증가함에 따라 연산량이 지수적으로 폭발합니다.
    * 사용자 데이터에서 $n=40$일 때 시간이 급증한 원인이 바로 이 지수적 증가 때문입니다.
