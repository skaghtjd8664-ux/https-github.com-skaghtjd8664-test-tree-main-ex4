# GCD알고리즘 성능 프로파일러

|입력값(a, b)|시간|
|------|---|
|18, 24|3.875초|
|256, 548|3.559초|
|1028, 15086|2.988초|

## 알고리즘 로직

두 수 $a, b$에 대하여 $a$를 $b$로 나눈 나머지를 $r$이라 할 때, $GCD(a, b) = GCD(b, r)$을 이용 $r=0$일 때의 $b$가 최대공약수

## 시간복잡도 

**Big-O:** $O(\log(\min(a, b)))$
* **분석 이유:** * 나머지 연산을 수행할 때마다 입력값의 크기가 매우 빠르게 감소합니다.
    * 가장 최악의 경우(연산 횟수가 가장 많은 경우)는 입력값이 **피보나치 수**일 때 발생하지만, 이조차도 로그 함수의 형태를 띱니다.
    * 사용자가 제공한 데이터에서 입력값이 커져도 실행 시간이 일정하게 유지된 이유는 이 효율적인 로그 복잡도 때문입니다.
